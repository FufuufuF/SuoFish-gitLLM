# SuoFish-gitLLM

SuoFish-gitLLM 是一个面向 AI 对话产品场景的项目，尝试把“聊天”从单线问答，扩展成可管理、可分叉、可回收的对话工作流。

它关注的不只是消息收发本身，而是当一次对话不断延展之后，用户如何切换主题、探索不同思路、保留上下文，并最终把分散的信息重新整理回主线。围绕这一目标，项目构建了会话列表、流式回复、线程分支树和分支合并等核心能力，让整个交互更接近真实的 AI 产品，而不是停留在演示性质的聊天页面。

## 项目亮点

- 多会话管理：支持在不同聊天会话之间快速切换
- 流式对话体验：AI 回复按流式方式逐步呈现，交互反馈更自然
- 对话分支能力：可以从当前线程继续分叉，形成清晰的分支树结构
- 分支合并能力：支持将子分支内容整理后合并回主线
- 上下文可视化：消息列表能够体现祖先消息与当前分支的关系
- 产品化界面：包含侧边栏、主题切换、骨架屏、分页加载等完整界面体验

## 核心功能

### 会话系统

项目支持多会话列表展示与切换，适合需要长期保留不同讨论主题的场景。新会话的创建和进入聊天流程无缝衔接，整体体验更接近真实产品而非演示级页面。

### 流式聊天

用户发送消息后，AI 回复会以流式方式逐步渲染。相比一次性返回完整文本，这种方式更适合展示实时生成体验，也更符合大模型产品的交互预期。

### 分支树对话

项目的一个核心特性是“线程分叉”。当某一段对话需要探索不同方向时，可以在当前线程上继续分出新的分支，而不是打乱原有上下文。侧边栏中的分支树能直观展示各条对话路径之间的关系。

### 分支合并

在不同分支完成探索后，可以将分支内容整理为简报并合并回目标线程。这使项目不仅能展示“发散式讨论”，也能展示“收敛式整理”的能力。

## 快速开始

```bash
pnpm install
pnpm dev
```

默认启动地址：

```text
http://localhost:8080
```

项目运行依赖本地后端接口，前端开发环境会将 `/api` 请求代理到：

```text
http://localhost:9090
```
